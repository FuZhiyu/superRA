## Objective

You get a correct-but-rough analysis reworked into a change that fits your project: local conventions matched, helpers you already have reused instead of re-implemented, stale project docs updated, and the diff pruned to the minimum net change so what lands is a clean, reviewable PR rather than a pile of one-off scripts. Reach for it when single-shot agent output is right but does not match the surrounding code.

The integration phase leans on it most, but you can invoke it any time you want a branch made codebase-coherent before review. See [`refactor-and-integrate`](skills/refactor-and-integrate/SKILL.md).
