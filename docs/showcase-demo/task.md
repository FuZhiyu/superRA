## Objective

Estimate whether firm size and price momentum predict the cross-section of next-month stock returns, using a fully simulated monthly equity panel so the whole project is public and reproducible. This is a teaching example: the data are generated by a known process, so the "right answer" is built in and every step can be checked against ground truth.

The workstream runs the full PLAN -> IMPLEMENT -> INTEGRATE loop on a tiny scope. Simulate the panel, construct the size and momentum sorting variables, estimate the size-sorted and momentum-sorted return spreads, check robustness with a double sort, and write up the findings. Children are dispatched along the dependency edges; statuses are mid-flight on purpose so this tree shows what an in-progress project looks like.

### Conventions

- Returns are monthly and simple (not log); the size variable is log market capitalization; momentum is the cumulative return over months $t-12$ to $t-2$ (skipping the most recent month).
- All data are simulated — there are no real tickers, firm names, or vendor records. Counts and "results" below are illustrative figures for a documentation demo, not findings about any real market.
- Portfolio sorts use value-weighted decile portfolios unless a task says otherwise; spreads are top-minus-bottom.

### Context

- The data-generating process plants a small positive size premium and a moderate momentum premium, so the estimated spreads should recover those signs. A task that recovers the wrong sign is a bug, not a finding.
- This tree is also the worked example the documentation Quickstart walks through, so the toy analysis here and the tutorial's narrative are meant to tell the same story.
