## Objective

Builds on the approved `05-recursive-context-conventions`, which made task context recursive (an agent reading a task receives its ancestor chain as context). Two refinements to how `task_read.py` renders that context:

1. **Rename the `=== Ancestor Context ===` section to `=== Context ===`.** The recursive model treats the whole render as the context an agent needs; the entire block above the task is context, so qualifying only the ancestor portion as "...Context" is the confusing part. "Context" is also what the code's own docstring/comments already imply.

2. **Make the task tree visible inside the Context block.** Today the Context section shows only the ancestor *spine* as text excerpts — the agent cannot see its position, siblings (only `depends_on` ones appear, lower down), or children. Add a compact **focused** tree at the top of the Context block: the path from the tree root down to the current task, with the current task's siblings and direct children shown and the current node marked. Do **not** expand the other branches of ancestor tasks — the shared root can be a large tree, and dumping all of it buries the signal. Reuse `task_query.py`'s existing tree renderer rather than writing a second one.

Target render shape (example, current task `task-tree/planning-redesign/review-planning-protocol/06-context-section-and-tree`):

```
=== Context ===
Review and Planning Protocol
  ...
  task-tree: Task Tree Skill
    planning-redesign: Planning Workflow Redesign
      review-planning-protocol: Review and Planning Protocol
    01-objective-guidance-task-anatomy
    ... (siblings)
    ▶ 06-context-section-and-tree: Context section rename + visible task tree   ← this task

<ancestor ## Objective excerpts, as 05 left them>

=== Task: ... ===
=== Sibling Dependencies ===
```

(Marker glyph and exact indentation are implementer's choice as long as the current node is unambiguously marked and the tree is the focused/bounded form above.)

## Results

Both refinements implemented and verified.

**Header rename.** [`task_read.py`](../../../../../skills/task-tree/scripts/task_read.py) human render now emits `=== Context ===` (was `=== Ancestor Context ===`). Ancestor-sibling branches are **not** expanded. `task_read.py` imports this helper (no duplicate rendering logic) and injects the tree at the top of the Context block, above the ancestor objective excerpts.
