## Objective

The known frontend performance and staleness defects from the 2026-07-19 review are fixed:

- The sidebar filter input is debounced and no longer re-scans each subtree once per ancestor on every keystroke.
- SSE task events trigger at most one debounced comment-summary refresh per burst, and badges are not torn down before the fetch resolves (no flicker window).
- The children-panel cache signature includes the title, and the cache is cleared on worktree switch and full-reload; `pathTitles` (breadcrumb titles) refreshes when an SSE row swap changes a title — net effect: a title edit propagates to the children panel and breadcrumb without a structural reload.
- The worktree dropdown issues one `/api/worktrees` fetch per open, not two.
- Dead parameters and never-invoked closures in the comment UI are removed; `style.cssText` strings duplicating existing CSS rules become classes.
- Validation: a targeted test (or scripted browser check) shows a title edit propagating to panel + breadcrumb live; one summary fetch observed per SSE burst; both suites green.

## Results

All six fixes plus both advisory items landed in [dashboard.js](../../../../skills/task-tree/scripts/templates/dashboard.js) and [dashboard.css](../../../../skills/task-tree/scripts/templates/dashboard.css); `base.html` was untouched (out of scope per the parallel-dispatch note, and unnecessary — the `oninput`/`onchange` attributes already call the global names the fix wraps).

- **Filter debounce + single-pass walk**: [dashboard.js:657-674](../../../../skills/task-tree/scripts/templates/dashboard.js#L657-L674) — `applyFilters()` is now a 150ms-debounced wrapper around the renamed `applyFiltersNow()`; `base.html`'s `oninput`/`onchange` attributes call the same global name unchanged. `applyFiltersToNode` ([dashboard.js:670-698](../../../../skills/task-tree/scripts/templates/dashboard.js#L670-L698)) was restructured into a single post-order pass: each node's own-row status/search match is evaluated once and combined bottom-up with its children's already-computed results, replacing the old `nodeMatchesStatus`/`nodeMatchesSearch` (each independently re-recursing a node's whole subtree, stacked on top of `applyFiltersToNode`'s own recursion into visible children — O(n²) worst case, a row visited once per ancestor). Both now-unused helpers were deleted. The bottom-up accumulation tracks status-match and search-match independently across the subtree before ANDing them for a node's own visibility, reproducing the prior semantics exactly (including the cross-descendant case: a branch shows as visible when one descendant alone satisfies the status pill and a different descendant alone satisfies the search text).
- **SSE comment-badge burst + no flicker**: [dashboard.js:2842-2848](../../../../skills/task-tree/scripts/templates/dashboard.js#L2842-L2848) adds a 150ms-debounced `scheduleTreeCommentBadgeRefresh()`, called from the `htmx:sseBeforeMessage` handler in place of the old per-event `setTimeout(updateTreeCommentBadges, 100)`. `updateTreeCommentBadges` ([dashboard.js:2847](../../../../skills/task-tree/scripts/templates/dashboard.js#L2847)) now clears `.tree-comment-badge` only after the fresh set is built, right before inserting it — old badges never disappear while the fetch is in flight.
- **Children-panel cache + pathTitles propagation**: `childrenSig` ([dashboard.js:1180-1187](../../../../skills/task-tree/scripts/templates/dashboard.js#L1180-L1187)) now folds the title into the signature. `_childrenDagCache` is cleared in `onFullReload` ([dashboard.js:2304-2313](../../../../skills/task-tree/scripts/templates/dashboard.js#L2304-L2313)) and `applyWorktree` ([dashboard.js:2527-2533](../../../../skills/task-tree/scripts/templates/dashboard.js#L2527-L2533)). `onTaskUpdate` ([dashboard.js:2274-2298](../../../../skills/task-tree/scripts/templates/dashboard.js#L2274-L2298)) was restructured so the deferred (post-swap) callback runs `indexNavTitles()` + `updateBreadcrumb(activePath)` *before* the sibling-card `loadChildrenDag(activePath)` call, not before it — ordering matters because the root-level children panel (`rootChildrenFromNav`) reads `pathTitles` directly rather than a server fetch, so starting it pre-reindex would bake the stale title into a freshly (mis-)cached render.
- **Worktree dropdown single fetch**: [dashboard.js:2569-2583](../../../../skills/task-tree/scripts/templates/dashboard.js#L2569-L2583) — `initWorktreeSelectorRefresh` coalesces the same-tick `mousedown`+`focus` pair (a mouse-driven open fires both) via a `pending` flag cleared on the next tick.
- **Dead params/closures + cssText-to-classes**: `resolveComment`/`deleteComment` drop the unused `btn` param ([dashboard.js:3021-3032](../../../../skills/task-tree/scripts/templates/dashboard.js#L3021-L3032)); `_activeCommentEdit`'s never-read `save`/`cancel` closures are gone. Every `style.cssText`/manual `style.*` assignment in the comment UI (comment-form buttons, edit-mode textarea/Save/Cancel, and the per-comment Edit/Resolve/Delete row) is now a class — `.comment-edit-textarea`, `.comment-edit-btn`(`.btn-secondary`), `.comment-actions`, `.comment-action-btn`(`--danger`) — defined once in [dashboard.css:1683-1724](../../../../skills/task-tree/scripts/templates/dashboard.css#L1683-L1724).
- **Advisory — parallel nav restore**: `restoreExpandedNavPaths` ([dashboard.js:1537-1550](../../../../skills/task-tree/scripts/templates/dashboard.js#L1537-L1550)) now groups paths by depth and runs each depth level's `expandNavNode` calls concurrently via `Promise.all`, awaiting only across depth levels (a child's lazy-loaded DOM row depends on its parent's expand having completed; siblings at the same depth do not depend on each other).
- **Advisory — completion-hook badge patch**: `setActive` now records `_lastSidebarUpdate = updateSidebar(path)` ([dashboard.js:806-810](../../../../skills/task-tree/scripts/templates/dashboard.js#L806-L810), [dashboard.js:848](../../../../skills/task-tree/scripts/templates/dashboard.js#L848)); `patchCardBadgeWhenReady` ([dashboard.js:1095-1113](../../../../skills/task-tree/scripts/templates/dashboard.js#L1095-L1113)) awaits that promise once instead of polling every 60ms up to 10 times.
