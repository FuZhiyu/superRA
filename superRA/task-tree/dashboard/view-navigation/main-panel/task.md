## Objective

Fill the main panel: the active node's own content, then its **direct children** as clickable cards. Load `frontend-design:frontend-design` before touching markup or CSS. Live server only; edit `skills/task-tree/scripts/templates/base.html`. Build on `routing-shell` (the `#active-node` / `#children-dag` regions and `setActive`) and `server-partials` (the `/node/{path}` route). This task and `sidebar-nav` both edit `base.html`, so they run serially ‚Äî coordinate edits to shared functions.

**Active-node card ‚Äî implement `loadActiveNode(path)`** (the stub from `routing-shell`): fetch `GET /node/{path}` into `#active-node`, then render its markdown sections and load comments by reusing the existing pipeline unchanged ‚Äî `renderMarkdown` ([`base.html:679`](../../../../../skills/task-tree/scripts/templates/base.html#L679)), the section render-on-reveal logic from `expandTaskDetails` ([`base.html:1105`](../../../../../skills/task-tree/scripts/templates/base.html#L1105)), and `loadComments(path)`. Sections default to **expanded** in the card (this is the focused detail view). Show the task's title + status badge at the top of the panel (the breadcrumb already shows the path).

**Children panel ‚Äî implement `loadChildrenDag(path)`** so a parent's **direct children always render as the clickable `.child-card` grid, never a mermaid graph.** Only the direct children are shown ‚Äî never recurse into grandchildren. Three cases:

- **Leaf** active node ‚Üí render nothing in `#children-dag` (card only).
- **Children with no inter-child dependency** ‚Üí the flat `.child-card` grid (slug + title + status badge per child), each `setActive(childPath)` on click. This is the existing `buildChildGrid` path ‚Äî keep it.
- **Children with at least one inter-child dependency** ‚Üí the **same cards**, arranged as a **layered dependency flow** (replacing the old mermaid render). Group the children into topological tiers ‚Äî tier 0 = children that depend on no sibling; tier *k* = children all of whose sibling-dependencies sit in earlier tiers ‚Äî stack the tiers top‚Üíbottom in execution order, and render each tier as a row of the same `.child-card`s. On any card that depends on a sibling, add a small footer line ‚Äî `‚Ü≥ after: <slug>[, <slug>‚Ä¶]` ‚Äî naming its **direct** sibling dependencies only (not transitive). Use a subtle inter-tier separator/arrow cue to signal flow direction; do **not** draw SVG node-to-node connectors. Cards stay clickable (`setActive`).

Fetch `GET /dag?root={path}` ([`plan_dashboard.py`](../../../../../skills/task-tree/scripts/plan_dashboard.py)) as the data source ‚Äî it already returns exactly the direct-children sibling graph. Parse the fragment client-side for (a) the child set + per-child status (from `data-node-paths` and the `style <id> fill:#<color>` lines, as today) and (b) the dependency edges from the `<dep_id> --> <child_id>` lines. **Edge direction is prerequisite ‚Üí dependent** (`{{ dep_id }} --> {{ child_id }}` in [`dag.html`](../../../../../skills/task-tree/scripts/templates/dag.html)), so `child_id` depends on `dep_id`; build the tiers and the `after:` labels from that. Make tiering cycle-safe: if a full pass places no remaining child (a cycle), drop the leftovers into the last tier rather than looping. Render the children panel **after** the card paints; keep the `(path, child-status-signature)` cache so revisiting an unchanged node restores the built HTML without recomputing.

**Remove mermaid entirely.** The children panel was the only remaining mermaid render; once it renders cards, mermaid is dead. Delete the `mermaid.run` call, `mermaid.initialize`, the mermaid `<script src=‚Ä¶>` CDN tag, the `.children-dag .mermaid` CSS, `wireDagNodeClicks`, and the now-orphaned mermaid comments. Keep `dag.html`, the `GET /dag` route, and the `node_id‚Üîpath` map ‚Äî they remain the parsed data source. The cache `kind` collapses to the single card/flow path. (`renderDagView` and `toggleDagPanel` were already removed in earlier tasks; the `revealTask`‚Üí`setActive` alias for Kanban cards stays.)

## Results

The main panel renders the active node's content card, then its **direct children as clickable `.child-card`s** ‚Äî a flat grid when the children have no inter-child dependency, the same cards in a **layered topological flow** when they do. Mermaid is removed from the page entirely. `loadActiveNode` and `loadChildrenDag` live in [`base.html`](../../../../../skills/task-tree/scripts/templates/base.html); the empty `#active-node` / `#children-dag` containers fill on every navigation. All work is on the live server template; the static `generate` path is untouched.

**Active-node card ‚Äî `loadActiveNode(path)`** ([base.html:1622](../../../../../skills/task-tree/scripts/templates/base.html#L1622)). Fetches the body-only `GET /node/<path>` partial and wraps it in a real `<div class="task-node active-node-body" data-path="<path>">` inside `#active-node`. That wrapper is the key to reuse: `toggleSection`, `loadComments`, `updateSectionBadges`, and the comment forms all resolve their task context via `.task-node[data-path]`, so the entire existing section + comment pipeline consumes the partial byte-for-byte as the old tree body did ‚Äî no fork of that code. A header (`.active-node-head`) shows the task's `slug ¬∑ title` (title via `textContent`, no injection) plus its status badge; the breadcrumb carries the path. Sections default to **expanded**: [`revealCardSection`](../../../../../skills/task-tree/scripts/templates/base.html#L1693) mirrors `toggleSection`'s expand branch (add `.open` to `.section-content`, `uncapAfterTransition`, lazy-render the `text/x-markdown` payload via the unchanged `renderMarkdown`) without the click event, scoped to `:scope > [data-section]` so the nested `.commentable-block`s that `renderMarkdown` injects are not re-revealed. A monotonic `_token` guard drops a stale fetch that lands after the user has navigated on. (This half was unchanged by the 2026-05-31 rework; its line cites are refreshed.)

**Status-badge timing.** The badge status is read off the sidebar row's `data-status` ([`navRowStatus`](../../../../../skills/task-tree/scripts/templates/base.html#L1719)). On a fresh deep descent the sidebar's ancestor-walk (`updateSidebar`, started in the same `setActive` tick) can outlast the `/node` fetch, so the row may not exist yet. The status read is moved after the `await`, and [`patchCardBadgeWhenReady`](../../../../../skills/task-tree/scripts/templates/base.html#L1727) polls briefly (‚â§10√ó60ms, token-guarded) to inject the badge once the row materializes. Verified the badge lands on deep-link, child-card-click, and breadcrumb descents.

**Children panel ‚Äî `loadChildrenDag(path)`** ([base.html:1758](../../../../../skills/task-tree/scripts/templates/base.html#L1758)). A single `GET /dag?root=<path>` fetch supplies the data; [`parseChildrenDag`](../../../../../skills/task-tree/scripts/templates/base.html#L1841) reads (a) the child set from the authoritative `data-node-paths` map, (b) per-child status from the `style <id> fill:#<color>` lines (the fixed color‚Üístatus map mirroring `dag.html`), and (c) the inter-child dependency edges from the `<dep_id> --> <child_id>` lines (a line-anchored `/^\s*(\S+)\s*-->\s*(\S+)\s*$/gm` match), mapped back through the node-id map into `edges[childPath] = [depPath‚Ä¶]`. Edge direction is prerequisite ‚Üí dependent, so `child_id` depends on `dep_id`. Three cases, **direct children only ‚Äî never recursed**:
- **Leaf** (no child nodes) ‚Üí `#children-dag` cleared, card-only.
- **No inter-child deps** ‚Üí the flat clickable `.child-card` grid via [`buildChildGrid`](../../../../../skills/task-tree/scripts/templates/base.html#L1924).
- **Inter-child deps** ‚Üí the same cards in a layered flow via [`buildChildFlow`](../../../../../skills/task-tree/scripts/templates/base.html#L1936).

**Layered dependency flow ‚Äî `buildChildFlow(children, edges)`.** Filters each child's deps to siblings actually present in the set, then greedily peels topological tiers: tier 0 = children with no remaining sibling-dep; tier *k* = children all of whose sibling-deps are already placed. **Cycle-safe** ‚Äî a pass that places nothing flushes all leftovers into the current (final) tier instead of looping. Each tier renders as a `.flow-tier` row of `.child-card`s; consecutive tiers are separated by a `.flow-sep` cue (a centered dashed vertical spine capped by a downward accent chevron ‚Äî a pure-CSS flow-direction signal, **no SVG node-to-node connectors**). Every dependent card carries a `‚Ü≥ after: <slug>[, <slug>‚Ä¶]` footer ([`.child-card-deps`](../../../../../skills/task-tree/scripts/templates/base.html#L1902)) naming only its **direct** sibling deps; dependent cards also get a faint `.has-deps` left accent. Both grid and flow share the single [`childCardHTML`](../../../../../skills/task-tree/scripts/templates/base.html#L1902) builder (slug + title + status badge, all escaped) and one delegated [`onChildCardClick`](../../../../../skills/task-tree/scripts/templates/base.html#L1971) ‚Üí `setActive` descent, so cached HTML stays clickable with no re-wiring.

The **root** is special-cased: `GET /dag?root=` with an empty root returns the *global* graph, not root's direct children, and the root never has inter-child deps ‚Äî so root's children are read straight off the always-present top-level nav rows ([`rootChildrenFromNav`](../../../../../skills/task-tree/scripts/templates/base.html#L1879)) and rendered as a grid. Output is cached in `_childrenDagCache` keyed by `path` with a [`childrenSig`](../../../../../skills/task-tree/scripts/templates/base.html#L1807) signature `(child-path:status:sorted-deps|‚Ä¶)`: revisiting an unchanged node restores the cached HTML, while a child whose **status or sibling deps** changed busts the signature and forces a re-render. The cache `kind` field is gone ‚Äî there is one card/flow render path now.

**Comment resolver disambiguation** (unchanged from the prior pass). Both the sidebar nav row and the active-node card carry `.task-node[data-path]` for the active task, sidebar row first in the DOM, so the global `document.querySelector('.task-node[data-path=‚Ä¶]')` in `loadComments`/`updateSectionBadges`/the `task:` SSE handler would resolve to the body-free nav row. [`commentTaskNode(taskPath)`](../../../../../skills/task-tree/scripts/templates/base.html#L2927) prefers `#active-node .task-node[data-path]` and falls back to the global match, so comments land where the commentable blocks live (the card).

**Mermaid removed.** Deleted the mermaid CDN `<script src=‚Ä¶>` tag, the `mermaid.initialize` call, the in-`loadChildrenDag` `mermaid.run` branch, the `wireDagNodeClicks` function, the `.children-dag .mermaid` and `.dag-node-clickable*` CSS, and the orphaned "DAG rendering (mermaid)" comments. No `mermaid` / `wireDagNodeClicks` / `onChildGridClick` reference remains in the rendered client JS (one descriptive comment notes the panel is *not* a mermaid graph). `dag.html`, the `GET /dag` route, and the `node_id‚Üîpath` map are kept as the parsed data source; `dag.html`'s header comment is updated to document the card-rendering role (it no longer references `wireDagNodeClicks` or an in-page mermaid render). `renderDagView` / `toggleDagPanel` were already gone from earlier tasks; the `revealTask`‚Üí`setActive` alias stays for Kanban cards.

**CSS** (editorial palette, theme-aware, all transitions auto-honor the existing global `prefers-reduced-motion` block): the `.child-grid`/`.child-card` cards (paper-card bg, terracotta accent slug, hover lift + accent border) are reused as-is; new are `.child-card-deps` (mono footer, accent `‚Ü≥`), `.child-flow`/`.flow-tier` (the tier stack and wrapping rows), `.flow-sep` (dashed spine + chevron), and `.child-card.has-deps`.
