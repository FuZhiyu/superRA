## Objective

Reduce the instruction surface a main agent loads in a typical superRA session (~16k words over a full plan→implement→integrate lifecycle today) by roughly 30% without weakening any gate. Four cut categories, reviewed with the researcher on 2026-07-01: (1) duplicated rules collapse to one authoritative home plus pointers; (2) necessity-test failures are deleted; (3) step-scoped procedure moves behind step-entry reference loads; (4) entry-time re-review of upstream-gated artifacts is removed — gates run once at the owning phase's exit, and runtime `BLOCKED`/`NEEDS-CTX` returns are the safety net for stale state.

Researcher decisions: `report-in-markdown` stays always-loaded (small and universally used); the `ensure-agent-orchestration` hook is narrowed so `superplan` no longer requires the companion load, but the gate stays for `superimplement`/`superintegrate`; `using-superra` §Code-Change Defaults stays in the body this round (adapter relocation deferred); `main-agent.md` §Surfacing the Live Dashboard keeps its inline deep-link quick-reference (2026-07-01), accepting the minor DRY overlap with `task-tree/SKILL.md` §Reading the Tree so the always-loaded reference works without a task-tree load.

Prior art: [main-agent-trimming](../main-agent-trimming/task.md) rewrote `main-agent.md` to the status+git resume model. This tree extends the same discipline to the whole load surface: `using-superra`, the three workflow skills, `agent-orchestration`, and the skill-load hook.

### Conventions

- The CLAUDE.md "Teach the Protocol, Don't Prescribe Each Action" gate applies line-by-line to every diff hunk; DRY + Necessity are verified per line by both implementer and reviewer.
- After moving or deleting any section, sweep inbound pointers repo-wide (`grep -rn "§<Section Name>"` over `skills/ agents/ docs/ hooks/ tests/ README.md CLAUDE.md`) and re-point or rewrite each hit; a dangling § pointer is a `[BLOCKING]` finding.
- Generated artifacts must not be hand-edited: `skills/using-superra/references/direct-mode-implementer.md`, `direct-mode-reviewer.md`, `.codex/agents/superra_implementer.toml`, `.codex/agents/superra_reviewer.toml` (generator: `skills/codex-superra-setup/scripts/sync_codex_agents.py`). No subtask here should need role-spec changes; if an edit would, regenerate instead of hand-editing.
